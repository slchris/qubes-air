调用路径

数据怎么流，网域怎么分

自上而下三层：① 本地控制域 → ② 网络出口链 → ③ 远端算力域。本地发起、经四级隔离出站、穿过 NAT 到远端执行，回程沿同一连接返回。

两个视角：蓝线是 sys-relay 发起的应用层调用（gRPC 双向流到远端 Zone，请求与回程同一条流）；灰链是这条流量的底层网络出口——所有出站都必须经 airnf → airnet → firewall → net 四级隔离才触外网。vault-cloud 全程无网；家庭 NAT 后无需公网入站端口，两侧 dom0 各校验一次。

为什么

Qubes 很安全，但被硬件困住了

找到兼容 Qubes 的硬件很难，本地资源也跑不动 GPU 训练或大编译。Qubes Air 的思路是：Qube 不必是本地 Xen VM，它可以是云上的 VM，也可以是一台物理分离的设备。

本地弱

笔记本只做控制中枢，几个轻量 qube 就够。

远端强

GPU、大内存、快网络放在云上，按需拉起。

用官方 RemoteVM 机制接回本地，而不是自造一套私有隧道协议。方向和官方一致，两侧 dom0 policy 双重校验。

工作原理

在控制台点一下，从建机到接回本地

底层是 Terraform、RemoteVM、qrexec，但你不用碰命令行。日常操作在控制台点按钮完成，命令行留给想深入的人。

建远程 Qube

填个表单就建好

选 Zone、规格、数据盘，点"创建"。控制台在后台编排 Terraform，计算与数据盘自动拆开。

自动接回本地

建好即经 RemoteVM 接回

本地 sys-relay 维持一条出站 gRPC 双向流长连接，qrexec 请求经它转发。反向调用走同一条流回程，无需公网入站。

一键挂起

不用时点"挂起"

点一下销毁计算实例、保留数据盘。下次点"恢复"挂回同一块盘，只付用到的机时。

存算分离

计算的钱只在真用时花，数据一直在

计算实例和数据盘是两个独立资源。挂起时销毁昂贵的计算实例，便宜的数据盘用 prevent_destroy 保留。恢复时几分钟拉起新实例、挂回原盘，数据原样回来。

  • + GPU 训练：只付周一那几小时，数据盘全月留存
  • + 数据盘用云侧 LUKS 加密，密钥只在本地
  • + 销毁即丢密钥，不依赖云盘上无效的擦除

如何落地

从本地 Qubes 起步，四步接上云

起点是你自己的 Qubes 笔记本，不是任何托管服务。云凭据放进本地无网络的 vault，永不上云，也不交给控制台。

  1. 1 本地初始化

    在 dom0 建三个 qube

    跑一次 init-qubes-air.sh，建好 mgmt-air（编排控制台）、sys-relay（出站回程）、vault-cloud（无网络凭据库）。

  2. 2 放云凭据

    AK/SK 存进 vault-cloud

    把 GCP 服务账号密钥、Proxmox API token、SSH 私钥放进 vault-cloud。它没有网卡，密钥出不去；用时经 qrexec 询问按需下发，不落盘。

  3. 3 建远程 Qube

    在控制台点"建"

    选 Zone、规格、数据盘，控制台在 mgmt-air 上编排 Terraform，在 Proxmox / GCP / AWS 上拉起实例，计算与数据盘自动拆开。

  4. 4 接回本地

    经 RemoteVM 回到桌面

    远程 Qube 经 RemoteVM 与 qrexec 接回本地，像本地 qube 一样用。家庭 NAT 后无需公网入站，两侧 dom0 各校验一次。

起点是本地部署，云只提供算力。你连的是自己的 Google / Proxmox 账号——凭据始终在你手里，控制台和云厂商都拿不到明文。

安全模型

不为功能牺牲安全

这套东西经过一轮对抗性设计评审重构。下面几条是硬红线，写进了代码和文档。

dom0 永远离线

编排全在联网的管理 qube，dom0 只经 Admin API 被动配合。这是 Qubes 的第一铁律。

私钥不出 vault

云 API key、SSH 私钥存无网络的 vault-cloud，用时经 qrexec 询问下发，不落盘、不上云。

远端到本地一律询问

qrexec policy 里，远端发起的调用永不自动放行，破坏性操作在 dom0 弹窗确认。Relay 不得直达 dom0。

云厂商不可信

state 用客户端加密后再进 backend，云只看到密文。不用云 KMS 去"防云厂商"，逻辑上讲不通。

诚实说明：无机密计算（如 AMD SEV-SNP）时，云运营商仍能读 VM 内存。因此云 Zone 当前只承载低敏感或一次性负载，高敏感任务留在本地或物理分离设备。
